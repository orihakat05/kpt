# 概要

KPT用のApplicaitonを作成するためのWebAPI


## 開発フロー

- Githubフローとし、masterがmain repositoryです
- PullRequestを出して開発を行う
- 追加コードには、それをカバーするRspecを必須とする


## 仕様について

- KPTはIssueモデルとして扱い、Keep, Problem, Statusをstatusとして持つ
